# Persona Battlecard — Salesforce Admin

## What they care about

- field mapping
- data cleanliness
- permissions
- user friction
- reliability
- admin burden
- duplicate creation risk
- support complexity

## Core pain

They do not want another messy integration that creates more cleanup work.

## Halosight angle

Structured capture can reduce manual entry and improve consistency if implemented with discipline.

## Strong approved message

Halosight should enhance Salesforce data quality and user adoption, not create another data mess.

## Common objections

- “What exactly writes into Salesforce?”
- “How are objects/fields mapped?”
- “Can users edit before save?”
- “How do you prevent bad data?”
- “What is the admin overhead?”

## Approved answers

Use only verified answers. If exact product behavior is unknown, use this safe framing:

- Field mappings, write logic, permissions, and governance need to be reviewed in implementation design.
- Our goal is controlled, structured insertion aligned to your current Salesforce architecture.
- We should validate object mappings, workflow triggers, and edit/approval rules with your admin team before deployment.

## Buying triggers

- rep noncompliance
- admins drowning in cleanup
- poor data completeness
- complaints from leadership about CRM trust
