# Persona Battlecard — IT

## What they care about

- security
- privacy
- access control
- data handling
- architecture
- vendor risk
- deployment burden
- compliance posture

## Core pain

They do not want an AI vendor creating security risk or integration fragility.

## Halosight angle

Position Halosight as controlled, scoped, enterprise deployment with clear data handling and phased rollout.

## Strong approved message

We know security scrutiny is part of the buying process. We aim to be transparent, disciplined, and implementation-minded.

## Common objections

- “What security certifications do you have?”
- “How is data stored?”
- “How is access controlled?”
- “What data is retained?”
- “What is your AI model architecture?”
- “Do you use customer data to train models?”

## Approved answers

Only use verified security facts.

Safe default framing:

- We can provide security and privacy documentation for formal review.
- We can walk through architecture, data handling, access model, and integration scope with your technical stakeholders.
- We should not make unverified claims about certifications, residency, encryption, retention, or model training.

## Never say unless verified

- SOC 2 Type II certified
- HIPAA compliant
- zero data retention
- customer data is never stored
- all data is encrypted end-to-end
- customer data is never used for training
- native support for any specific SSO / IAM provider
- exact deployment architecture
